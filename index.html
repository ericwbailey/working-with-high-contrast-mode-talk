The three Microsoft vendor-prefixed media queries


                        /* Targets the Windows High Contrast mode Black theme: */
                        @media (-ms-high-contrast: white-on-black) { }

                        /* Targets the Windows High Contrast mode White theme: */
                        @media (-ms-high-contrast: black-on-white) { }

                        /* Targets any active Windows High Contrast mode theme */
                        @media (-ms-high-contrast: active) { }

The one usable Microsoft vendor-prefixed media querie


                        /* Targets the Windows High Contrast mode Black theme: */
                        @media (-ms-high-contrast: white-on-black) { }

                        /* Targets the Windows High Contrast mode White theme: */
                        @media (-ms-high-contrast: black-on-white) { }
                        
                        /* Targets any active Windows High Contrast mode theme */
                        @media (-ms-high-contrast: active) { }

Forced color mode query


                        /* Targets any active Windows High Contrast mode theme */
                        @media (-ms-high-contrast: active) { }

                        /* Targets any active Forced Colors mode theme */
                        @media (forced-colors: active) { }

Notes

  • You can target both Windows High Contrast Mode and Forced Colors mode
  • Be careful about IE support concerns and survivor bias

Activate High Contrast Mode


                        /* Apply styling when High Contrast Mode is active */
                        @media (-ms-high-contrast: active) {
                        }

Apply a border


                        /* Apply styling when High Contrast Mode is active */
                        @media (-ms-high-contrast: active) {
                            input {
                                border: 2px solid #ffffff;
                            }
                        }

Notes

  • Take all input tags and give them a border
  • Trouble with this is colors are absolute, and themes are dynamic
  • so a white border won't be visible on a black text on white background mode
  • The same applies for any other color

Apply a color keyword to the border


                        /* Apply styling when High Contrast Mode is active */
                        @media (-ms-high-contrast: active) {
                            input {
                                border: 2px solid buttonFace;
                            }
                        }

Notes

  • What we can do is assign it a keyword
  • buttonFace will dynamically update with the theme

Target ARIA using a CSS attribute selector and style


                        <span class="button" role="button" tabindex="0">
                          Download Pizza
                        </span>
                    

                        /* Apply styling when High Contrast Mode is active */
                        @media (-ms-high-contrast: active) {
                            [role="button"] {
                                border: 0.15em solid buttonText;
                                color: buttonText;
                            }
                        }
  Download Pizza  

Notes

  • Attribute selector that targets role of button
  • Manipulate the border and the color to make something recognizable as a button in HCM
  • I personally find that rounded edges help, they're a great affordance to communicate something is buttony
  • Be careful: Gradients are technically background images and Edge has opinions on this, more on that in a bit

Content considerations

Background images

“Don’t use CSS background images to deliver content.”

—Terrill Thompson

Edge generates backgrounds for text over background images

Notes

  • It isn't visually very pleasing, but it is legible
  • I don't know about you, but I'd rather have something ugly and usable over something pretty but illegible any day of the week

Raster images

JPEGs, PNGs, GIFs, etc.

Notes

  • Images described with pixels
  • They are static, which means the browser does not know what content they contain
  • So it is important to assign them alt descriptions

Non-interactive icons

                        /* Non-interactive icons */
                        .icon-svg {
                          fill: Text;

File saved!

Notes

  • Note that this cascades to match text color and you don't have to do extra work

Button icons


                        /* Interactive icons */
                        @media screen and (-ms-high-contrast: active) {
                          button.icon-svg {
                              background: none;
                              border-color: buttonText;
                              fill: buttonText;
                          }
                        }

Styling SVG with CSS


                        /* Static color declarations */
                        .bar-chart { background-color: #ffffff; }

                        .bar-chart .frame { stroke: #000000; }

                        .bar-chart .text { fill: #000000; }

                        .bar-chart .age-sixtyfour-under { fill: #0074d9; }

                        .bar-chart .age-sixtyfour-under { fill: #ffdc00; }

Dynamically styling SVG with CSS in High Contrast Mode


                        /* Dynamic color when High Contrast mode is active */
                        @media (-ms-high-contrast: active) {
                          .bar-chart { background-color: window; }

                          .bar-chart .frame { stroke: highlightText; }
                          .bar-chart .text { fill: highlightText; }

                          .bar-chart .age-sixtyfour-under { fill: windowText; }

                          .bar-chart .age-sixtyfour-over { fill: highlight; }
                        }

Fancy tricks

CSS that removes content from High Contrast Mode


                        /* Remove from High Contrast Mode, but maintain layout */
                        @media (-ms-high-contrast: active) {
                            .hcm-remove {
                                visibility: hidden;
                            }
                        }

CSS that forces content to display as text in High Contrast Mode


                        /* Force content to display as text in High Contrast Mode */
                        @media (-ms-high-contrast: active) {
                            .hcm-display-as-text {
                                color: windowText;
                            }
                        }

CSS that forces a border


                        /* Draw an invisible border that renders when HCM is active */
                        .button-without-border {
                            border: 3px solid transparent;
                        }
                        /* Credit to Joe Watkins @_josephwatkins */

Cut the Microsoft mustard


                        if (window.matchMedia("(-ms-high-contrast: active)").matches) {
                          // We have support for IE 10+
                        } else {
                          // Load a good fallback experience
                        }

Notes

  • Be careful about device sniffing
  • Could be handy for environments like enterprise
  • Where granular Microsoft support is an unfortunate necessity
